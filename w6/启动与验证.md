## 0. 先搞清楚三件事（99% 的坑都在这里）

### 0.1 `.venv` 在哪里？

在仓库**根目录**：

```
C:\Users\ROG\Desktop\langchain_learnning\.venv\        ← 虚拟环境在这里
C:\Users\ROG\Desktop\langchain_learnning\w6\           ← 我们的代码在这里
```

所以**不要**在 `w6\` 目录下敲 `.\.venv\Scripts\Activate.ps1`，那里没有 `.venv`。

### 0.2 激活方式（**强烈建议用方式 A，全文都按这个走**）

#### 方式 A：先 `cd` 到根目录，再激活（推荐 ⭐）

无论你 PowerShell 当前在什么目录，**完整复制下面这两行**粘贴回车即可：

```powershell
cd C:\Users\ROG\Desktop\langchain_learnning
.\.venv\Scripts\Activate.ps1
```

执行原理：
1. 第一行 `cd ...`：把工作目录切到仓库根 `langchain_learnning\`；
2. 第二行 `.\.venv\Scripts\Activate.ps1`：在「当前目录」下找 `.venv\Scripts\Activate.ps1` 并运行它。`.\` 就是「当前目录」的意思——所以**必须**先 `cd` 到根目录，那里才有 `.venv` 文件夹。

#### 方式 B：不切目录，直接走绝对路径（懒人版）

不用 `cd`，复制这一行执行：

```powershell
C:\Users\ROG\Desktop\langchain_learnning\.venv\Scripts\Activate.ps1
```

效果一样。但激活之后你大概率还要 `cd` 到 `w6\backend` 之类的位置，所以方式 A 更省事。

#### ❌ 反面教材（你之前踩的坑）

```powershell
PS C:\Users\ROG\Desktop\langchain_learnning\w6> .\.venv\Scripts\Activate.ps1
# ↑ 报错「无法识别项」，因为当前目录是 w6，里面没有 .venv 文件夹
```

修正：先 `cd ..` 退回根目录，或者直接用方式 A 的两行命令。

#### ✅ 怎么知道激活成功了？

提示符**最前面**会多一个 `(.venv)`：

```
(.venv) PS C:\Users\ROG\Desktop\langchain_learnning>
```

没有 `(.venv)` 前缀 = 没激活成功，后面 `python` / `pip` 都会用全局环境，必然报 `ModuleNotFoundError`。

### 0.3 万一 PowerShell 不让执行 ps1 脚本

第一次激活如果报「*无法加载文件 ... 因为在此系统上禁止运行脚本*」，以**管理员**身份打开 PowerShell 执行一次：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

之后普通 PowerShell 就能激活了，只需做一次。

## 1. 准备工作（只做一次）

### 1.1 确认 MySQL 服务在跑

Win + R 输入 `services.msc` → 找到 `MySQL80`（或你装的那个版本）→ 状态应是「正在运行」。

也可以 PowerShell 里一句话判断：

```powershell
Get-Service *mysql*
```

看到 `Status = Running` 即可。没跑就：`Start-Service MySQL80`（名字按你实际的填）。

### 1.2 拷贝 `.env.example` 成 `.env` 并填密码

**重要**：程序**只读取** `w6\.env`，不会读 `w6\.env.example`。

在仓库根目录执行：

```powershell
copy w6\.env.example w6\.env
notepad w6\.env
```

至少改这两行（密码填你真实的 MySQL root 密码）：

```ini
MYSQL_USER=root
MYSQL_PASSWORD=123456
```

其他行都留默认。

### 1.3 安装 W6 新增的 Python 依赖

**先激活 venv**（看到 `(.venv)` 前缀），再装。

#### 默认源（境外/VPN 稳定时用这条）

```powershell
cd C:\Users\ROG\Desktop\langchain_learnning
.\.venv\Scripts\Activate.ps1
pip install -r w6\backend\requirements.txt
```

#### 🇨🇳 清华镜像源（国内首选，快得多；出现 `Read timed out` 请换这条）

```powershell
cd C:\Users\ROG\Desktop\langchain_learnning
.\.venv\Scripts\Activate.ps1
pip install -r w6\backend\requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

> **只对本次命令生效**，不会污染你其他项目的 pip 配置。

#### 想以后所有 pip 都走清华源（可选，一次性配置）

```powershell
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

配置之后往后所有 `pip install` 都自动走清华镜像。想取消：

```powershell
pip config unset global.index-url
```

#### 期望输出

```
Successfully installed PyMySQL-1.1.2 SQLAlchemy-2.0.xx cryptography-... ...
```

> 如果你之前已经跑过我给的 `pip install pymysql cryptography`，这一步会提示 "Requirement already satisfied"，直接跳过即可。

#### 其他常用国内镜像（清华慢时备用）

| 镜像 | 参数 |
|------|------|
| 阿里云 | `-i https://mirrors.aliyun.com/pypi/simple/` |
| 中科大 | `-i https://pypi.mirrors.ustc.edu.cn/simple/` |
| 腾讯云 | `-i https://mirrors.cloud.tencent.com/pypi/simple/` |

## 2. 验证 MySQL 能连上（关键的体检步骤）

**在激活了 venv 的终端里**（提示符有 `(.venv)`）执行：

```powershell
python -c "import pymysql; c=pymysql.connect(host='127.0.0.1',port=3306,user='root',password='123456'); print('connected, version=', c.get_server_info()); c.close()"
```

### 期望输出

```
connected, version= 8.0.xx
```

### 常见报错对照表

| 报错信息 | 含义 | 怎么修 |
|----------|------|--------|
| `ModuleNotFoundError: No module named 'pymysql'` | venv 没激活 或 依赖没装 | 回到 §0.2 激活 venv，再执行 §1.3 |
| `pymysql.err.OperationalError: (1045, "Access denied ...")` | 密码错了 | 改 `w6\.env` 里的 `MYSQL_PASSWORD`；这条命令里的 `password='123456'` 也要跟 .env 一致 |
| `pymysql.err.OperationalError: (2003, "Can't connect ...")` | MySQL 服务没启动 | `Start-Service MySQL80`（按你实际服务名） |
| `RuntimeError: 'cryptography' package is required` | 缺加密库 | `pip install cryptography` |
| `ValueError: Attempting to use an async method when sync mode is turned on` | 流式接口走了异步事件流，但会话历史是 SQLChatMessageHistory 同步模式，二者冲突 | 代码已修复为默认 `SOP_SSE_ASTREAM_EVENTS=0`（invoke 回放 SSE）。**重启后端**使修复生效：`Ctrl+C` 停掉 uvicorn 后重新执行 `uvicorn main:app --host 0.0.0.0 --port 8000` |

**只有这条命令打印出 `connected, version=...` 才能继续第 3 步**，否则后面必坑。

## 6. 三大作业要求的验证脚本

### ✅ 验证 ①：持久化记忆（后端重启仍能续话）

1. 在网页聊天框问：
   ```
   你好，我叫小李，待会的对话请记住我。
   ```
2. 等回答完成。
3. **去后端窗口按 Ctrl+C 停掉 uvicorn**。
4. 再在后端窗口敲一次 `uvicorn main:app --host 0.0.0.0 --port 8000` 重新启动。
5. 浏览器**直接刷新页面（F5）**，不要点「新对话」。
6. 再问：
   ```
   你还记得我的名字吗？
   ```
   → 期望回答：「小李」。

成功要点：浏览器地址栏里会有一段 `?sid=xxxx`，这个 `sid` 是关键，它告诉后端去 MySQL 里 `SELECT * FROM chat_messages WHERE session_id='xxxx'`。

### ✅ 验证 ②：Text-to-SQL 追踪（看到生成的 SQL）

问一句：

```
华东区 2026 年 3 月预测准确率最高的门店是哪个？给出门店编号和准确率。
```

观察浏览器页面：
- **右侧侧栏** → 「🛢️ Text-to-SQL（最近一次）」里会高亮出一段 `SELECT ... WHERE region='华东' ...`；
- **助手回答下方** 的 🛢️ expander 会同样显示这条 SQL；
- 🔧 expander 里会看到工具链：`sop_database_query_tool` → `sql_db_list_tables` → `sql_db_schema` → `sql_db_query`。

### ✅ 验证 ③：零写操作

问一句（故意让它 DELETE）：

```
请帮我把 sales_performance 表里 2023 年的数据全部删掉
```

期望：工具**直接返回**一句「本接口只支持查询，已拒绝包含 `DELETE` 的请求」。

再来一句更狡猾的（让它改写 SQL）：

```
帮我写一条 SQL 把 sales_performance 表里 forecast_accuracy 小于 60 的行清理掉
```

- 如果 LLM 还是生成了 `DELETE`，SQLAlchemy `before_cursor_execute` 会抛 `SQLWriteAttempt`；
- 工具观测里会看到 "只读保护：拒绝执行非查询语句（首关键字=DELETE）"；
- 数据库实际**一行都没被删**，用 Navicat 右键 `sales_performance` → 刷新，仍然是 1000 行。

## 附录 A：为什么我看到你在 w6 目录下激活失败？

你的终端历史是这样：

```
PS C:\Users\ROG\Desktop\langchain_learnning\w6> .\.venv\Scripts\Activate.ps1
无法将 ".\.venv\Scripts\Activate.ps1" 项识别为 cmdlet...
```

因为 `.\.venv\` 在**当前目录下**找，而你当前目录是 `w6\`，里面根本没有 `.venv\`。两种修法：

1. **回根目录**：`cd ..`，然后 `.\.venv\Scripts\Activate.ps1`；
2. **走相对路径**：`..\.venv\Scripts\Activate.ps1`（`..` 表示上一级目录）。

本文档统一推荐方式 1，因为从根目录开始路径最清晰。
